Documentación completa

Myoboku es una colección de recursos creados sobre el framework Twitter Bootstrap. En sí mismo, no es un framework, sino que es una "ampliación" de Bootstrap con un uso simplificado para cualquier programador con bases de HTML y CSS, ya que ni siquiera hace falta saber SASS, el lenguaje en el que está programado. Con un poco de conocimiento de programación (php, js, etc...), Myoboku se convierte en una herramienta poderosa para personalizar webs en segundos.

¿Por qué Myoboku no es un framework?

  • Al estar construído sobre el CDN de bootstrap, el estilo de Myoboku está preparado para sobreescribir el estilo de bootstrap en cualquier web, incluídos algunos temas o plugins de wordpress que traen bootstrap instalado por defecto.
  • Bootstrap 3 es uno de los frameworks más usados del mundo, por el tiempo que lleva desarrollado y por su simplicidad y potencia para construir webs rápidamente. Precisamente por esto, Myoboku marcará la diferencia notablemente con otras web en bootstrap sin apenas esfuerzo. Debido al tiempo que lleva desarrollado y al uso masificado, el estilo de bootstrap, aunque sigue siendo funcional, se ha vuelto un poco rancio gráficamente. Myoboku corrige este desperfecto.
  • Aunque bootstrap es bastante completo, es antiguo, y hay ciertos detalles que debe tener (botones mejorados, tablas responsive...). Myoboku incorpora estos detalles para que no tengas que estar buscando trozos de código en stackoverflow.

Preparando SASS

SASS es un precompilador de CSS enriquecido que trabaja en local. Sigue estos pasos para configurar un entorno en GULP que te permita editar los archivos y ver los cambios en el navegador gracias a browser-sync

  1. Instala node.js
  2. Abre la consola y sitúate en la carpeta donde vayas a editar Myoboku con cd seguido de la ruta de la carpeta
  3. Instala NPM siguiendo los pasos de su web
  4. Abre el archivo gulpfile.js en la raiz de Myoboku e instala las dependencias necesarias usando el comando de la consola npm install --save-dev(ejemplo npm install --save-dev gulp) 
    /*
* Dependencias de gulp
*/
var gulp = require('gulp');
var concat = require('gulp-concat');
var uglify = require('gulp-uglify');
var browserSync = require('browser-sync').create();
var sass = require('gulp-sass');
var reload = browserSync.reload;
  5. Ahora, configura el origen que va a reproducir el navegador en el archivo gulpfile.js (línea 11):
    • Si estás trabajando con XAMPP u otro programa de server local, en la tarea del server debes comentar la línea de server y descomentar la línea de proxy añadiendo la ruta de tu proyecto en XAMPP
    • Si estás trabajando en una carpeta normal, comenta la línea de proxy y descomenta la lína de server, que ejecutará la raíz de la carpeta indicada antes.
  6. Ya lo tienes todo configurado. Solo tienes que volver a la consola y ejecutar en el directorio base el comando gulp. Automáticamente se activará todo y se abrirá una ventana de navegador con tu proyecto. Si editas cualquier archivo html, php o scss, el cambio se recargará automáticamente en la ventana del navegador. Ojo, las librerias de Myoboku están alojadas en una carpeta llamada /sass/lib, cuando quieras cambiar algo en ellas, debes guardar el archivo y luego guardar el /sass/master.scss dentro de la carpeta /sass
  7. El archivo resultante de Myoboku se guardará en /css/master/master.css. Este es el archivo que debe leer tu html después de la llamada al estilo de bootstrap para poder sobreescribirlo correctamente.

Variables de Myoboku

Variables principales

Las variables de Myoboku estandarizan las partes más importantes del estilo partiendo de la base de bootstraps para una buena complementación.

La variable $auric

Esta variable determina todo el tamaño de la web, ya que todas las medidas en PX se han seteado con ella. Para partir de la base de Bootstrap, establece un valor de 15px.

Las variables de color

La paleta de colores de Myoboku contiene 5 colores de marca más los 3 colores de aviso establecidos por Bootstrap (danger, warning y success)

  • $main es el color principal de la marca
  • $cta debe ser el color más llamativo de los 5 para llamar la atención
  • $sup este color debe estar a juego con las dos variables anteriores, pero no destacar sobre ninguna de ellas
  • $trans está pensado para ser un gris claro que sirva al diseñador para romper la monotonía del blanco
  • $base este color es el negro de la web, el diseñador puede probar con varios tonos de negro

Además Myoboku tiene las variables $pale (white) y $nocolor (transparent), muy necesarias a la hora de diseñar.

Variables secundarias

Variables tipográficas

La variable $icon-font establece la familia de fuentes que contiene los iconos, por defecto es FontAwesome. La variable $font-family establece la tipografía de la web. Recuerda que debes cargar la fuente en el HTML si usas fuentes de Google Fonts.

Variables de luz

Como verás más adelante, los colores de Myoboku crean por sí mismos versiones oscuras y claras de sí mismos almacenados en clases. La cantidad de oscurecimiento y claridad que tienen esas versiones se determina con las variables $darking y $lighting que por defecto tienen un valor de 25%

Variables de z-index

La propiedad z-index puede generar confusiones cuando se trabaja con varios elementos con diferentes position. Myoboku crea 4 variables para controlar el eje Z: $lowindex, $medindex, $superindex y $megaindex.

Clases para espaciar

Cada una de las 4 clases de tamaño importa el mixin que establece un valor en forma de padding superior/inferior o izquierdo/derecho. Este valor se calcula sobre la variable $auric, multiplicando por un valor numérico que hace la diferencia entre cada clase.

@mixin v-spacing($value){
    padding-top: $value;
    padding-bottom: $value;
}
@mixin h-spacing($value){
    padding-left: $value;
    padding-right: $value;
}

//Vertical padding
.tiny {
  @include v-spacing($auric / 2);
}
.low {
  @include v-spacing($auric);
}
.med {
  @include v-spacing($auric * 2);
}
.up {
  @include v-spacing($auric * 4);
}

//Horizontal padding
.thin{
  @include h-spacing($auric / 2);
}
.slim{
  @include h-spacing($auric);
}
.fat{
  @include h-spacing($auric * 2);
}
.dope{
  @include h-spacing($auric * 4);
}

Colores

Para crear o editar colores de la paleta, primero hay que añadir el color como variable en el archivo /lib/_variables.scss siguiendo la estructura marcada + info. Luego hay que añadir dicha variable al mapa o "array" de colores que está en el mismo documento:

$array_color :
  (nocolor, $nocolor),
  (pale, $pale),
  (main, $main),
  (cta, $cta),
  (sup, $sup),
  (trans, $trans),
  (base, $base),
  (danger, $danger),
  (warning, $warning),
  (success, $success);

Cuando lo tengas listo, el archivo /lib/_color.scss creará automáticamente las clases para colorear fondos y/o textos de elementos html; además, creará las mismas clases con los mismos colores en sus variantes claras y oscuras. Esta variación la determinan las variables $darking y $lighting. Para fabricar las clases.

¿Cómo funciona el archivo _color.scss?

Primero, necesitamos declarar el mixin que creará las clases en el CSS final. Tiene 2 valores: uno para "imprimir" el nombre del color en la declaración de la clase ($name) y otro para traer el código del color almacenado en su variable ($color):

//Constructor
@mixin build-color-printer($name, $color){
    
    .#{$name}-text{
        color: $color !important;
    }
    .#{$name}-bg{
        background-color: $color !important;
    }
}

Luego, programamos el condicional para que tenga un tercer valor llamado $matrix. Este valor solo puede tener 3 valores: light para la variación clara, dark para la oscura o def (default) para no tener variación.

Si el valor es dark o light, este condicional añadirá al nombre del color el prefijo de su variación + un guión, es decir, dark- o light-. Dentro de esta condición, la variable $color que guarda el color del array, se aclara u oscurece en base a las variables globales $darking y $lighting.

//Condicional para variantes tonales
@mixin build-color($name, $color, $matrix){
    
    @if $matrix == 'def'{
        @include build-color-printer($name, $color)
    } @else{
        $name : $matrix + '-' + $name; 
        
        @if $matrix == 'dark'{
            @include build-color-printer($name, darken($color, $darking))   
        } 
        @if $matrix == 'light'{
            @include build-color-printer($name, lighten($color, $lighting))   
        }
    }
    
}

Por último, necesitamos ejecutar el constructor con el condicional haciendo que el array de colores mande cada uno de sus colores. Lo ejecutamos 3 veces para fabricar las clases nomales, las oscuras y las claras.

//Ejecutando el constructor con el array_color
@each $name, $color in $array_color{
    @include build-color($name, $color, 'def');
    @include build-color($name, $color, 'dark');
    @include build-color($name, $color, 'light');
}
